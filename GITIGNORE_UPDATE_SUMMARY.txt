## Security Note

CRITICAL: The .env file was previously tracked by git and contained sensitive information:
- Supabase URL
- Supabase Publishable Key
- Supabase Project ID

While these have been removed from tracking, they may still exist in git history.

### Recommended Actions for Production:

1. If this repository is public or will be shared:
   - Rotate all API keys in .env
   - Generate new Supabase anon key
   - Consider using git-filter-repo to remove .env from history

2. If this repository is private and controlled:
   - Keep current keys
   - Ensure .env stays untracked going forward
